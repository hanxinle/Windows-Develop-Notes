### 4. Visual Studio IDE 中编译操作说明

**推荐方式：一个空项目，逐个替换源文件**

**步骤 1：创建空项目**
- 打开 Visual Studio
- `文件` → `新建` → `项目`
- 搜索 `空项目`，选择 **"空项目"**
- 项目名称填 `CrashDemo`，位置选 `M:\20260504-CPP-crash\`，点击创建

**步骤 2：添加源文件**
- 在解决方案资源管理器中，右键 `源文件` → `添加` → `现有项`
- 选择 `crash_demos` 目录下的 `01_veh.cpp`
- （每次换 demo 时，先右键删除旧的 .cpp，再添加新的）

**步骤 3：设置编译选项（关键）**

右键项目 `CrashDemo` → `属性`：

| 路径 | 设置项 | 值 |
|------|--------|-----|
| C/C++ → 常规 | 警告等级 | `等级4 (/W4)` |
| C/C++ → 常规 | 将警告视为错误 | `否` |
| C/C++ → 代码生成 | 启用 C++ 异常 | `是 (/EHsc)` |
| C/C++ → 代码生成 | 运行库 (Debug) | `多线程调试 DLL (/MDd)` |
| C/C++ → 代码生成 | 运行库 (Release) | `多线程 DLL (/MD)` |
| 链接器 → 调试 | 生成调试信息 | `是 (/DEBUG)` |
| 链接器 → 输入 → 附加依赖项 | 附加依赖项 | 添加 `DbgHelp.lib` |

**步骤 4：编译运行**
- 按 `Ctrl+F5`（开始执行不调试）或 `F5`（调试）

**不同 demo 的特殊设置**：
- 01、02、03：必须链接 `DbgHelp.lib`
- 07：需要 C++17 标准（`C/C++ → 语言 → C++语言标准` → `ISO C++17 标准 (/std:c++17)`）
- 14：需要把 `14_cross_module_dll.cpp` 单独编译为 DLL 项目

**其他编译方式**：
- **CMake 方式**：VS → `打开本地文件夹` → 选择 `crash_demos`，VS 自动识别 CMakeLists.txt
- **命令行方式**：打开 "x64 Native Tools Command Prompt for VS"，用 `cl` 命令编译
